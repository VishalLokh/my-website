Automating Website Deployment with AWS CLI and GitHub

In this project, I created a fully automated process for deploying and updating my personal website on AWS. Rather than using a CDN, I utilized AWS CLI and a custom Bash script to streamline the entire deployment process from my local machine to the AWS cloud. The key steps involved are:

  • Website Development: I first developed the website locally using HTML, CSS, and other static files. This included creating an index.html page, along with associated stylesheets and images.
  • AWS S3 Bucket Setup: I hosted the website using an AWS S3 bucket configured for static website hosting. The S3 bucket was set up to store the website files and make them accessible to users.
  • Automation with AWS CLI: To ensure a smooth and continuous deployment process, I created a Bash script using AWS CLI commands. This script automates the upload of updated files from my local device directly to the S3 bucket, eliminating the need for manual updates.
  • Version Control with GitHub: I used GitHub for version control, allowing me to track changes to the website’s codebase. Any updates made to the local code are committed to a GitHub repository, ensuring version history and collaboration support.
  • Continuous Deployment: Whenever I make changes to the website locally, I run the Bash script to automatically push updates to the AWS S3 bucket, ensuring the website is always up-to-date without manual intervention.

This project showcases my ability to use cloud-based tools like AWS, along with Bash scripting and GitHub for automation, to create an efficient and scalable workflow for website deployment.
